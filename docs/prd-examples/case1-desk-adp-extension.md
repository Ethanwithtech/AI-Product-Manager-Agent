# PRD 风格参考案例 — Desk 扩展中心 ADP 集成

> 此文件作为 PM Strategist Agent 的 PRD Style Learning 案例。Agent 在生成需求文档时，会提取本案例的结构、语气、详细程度和格式特征，模仿输出。

## 案例来源
- 产品线：TCCC Desk 智能客服
- 需求类型：管理端功能开发（扩展中心 + 机器人管理）
- 特征：多期迭代规划、详细界面截图、交互逻辑描述、中英文翻译表

## 提取的风格特征（Style Profile）

```
文档结构：需求背景 → 需求描述(按诉求分块) → 界面清单(名称+截图+交互) → 翻译对照表
语气：技术产品文档风格，直接简洁，用"支持xxx"、"需要xxx"句式
术语：ADP、Desk、机器人、扩展中心、白名单、旗舰版、会话配置、渠道接入
详细程度：每个界面有独立描述+截图+交互说明，约2000-4000字/诉求
特殊格式：
  - 界面清单表格（界面名称 | 截图 | 交互/功能）
  - 中英文翻译对照表
  - 多期规划标注（本期/二期/三期）
  - "本期不做"显式标注
  - 存量用户逻辑单独说明
  - 弹窗文案逐字定义
优先级标记法：本期 > 二期 > 三期（无P0/P1标签，用期数区分）
```

## 原始需求单内容

### 一、需求背景

ADP是智能体机器人，不提供人工客服的功能。
存在客户对接Desk人工客服 + ADP智能体机器人的场景，故需要支持desk集成ADP——当前接口层面已开发完成，需管理端支持客户配置。
扩展智能客服Desk开放性能力。

### 二、需求描述

#### 诉求1（设计需求）

**本期**：
1. 支持用户同时创建ADP和自建机器人两种类型
2. 只支持一个类型下创建一个机器人

**存量用户逻辑**：存量逻辑还是走原来的页面（开发者设置页面），不支持开启扩展中心页面（不显示扩展中心页面路由），回调页面保持在开发者设置里面。

**二期**：支持配置多个会话配置，支持所有渠道绑定不同会话配置
**三期**：上线多客服号，允许启用多个机器人

#### 界面清单

| 界面名称 | 交互/功能 |
|---------|----------|
| 导航栏 | 工作台设置移到前往工作台按钮下面（本期不做），扩展中心放到快速开始按钮下面 |
| 开发者设置 | 删除对接第三方机器人选项，把回调URL放到回调类型选项下面 |
| 扩展中心主页 | 页面标题: 扩展中心，卡片：腾讯智能体开发平台、自建机器人 |
| 添加机器人-ADP | 弹窗配置：机器人名称+Appkey+SecretId+SecretKey |
| 添加机器人-第三方 | 弹窗配置：机器人名称+回调URL |
| 保存与启用 | 启用第三方会停用内部机器人，互斥逻辑 |
| 编辑/删除 | 编辑配置、删除确认弹窗 |

#### 关键业务规则
- 扩展中心为高级版功能
- 连接ADP为旗舰版功能（需白名单）
- 同时只能启用一个机器人（ADP/自建/内置互斥）
- 停用第三方机器人后，内部机器人不会自动恢复启用
- 配置名称字符长度和任务流名称长度限制保持一致
