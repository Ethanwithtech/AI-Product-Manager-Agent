# 多角度评估

## 概述

多角度评估技能提供一套结构化的分析框架，从五个互补的专业视角对输入的方案进行全面评估。通过模拟不同角色的思维方式，帮助发现盲点、平衡考量，最终得出客观、全面的综合结论。

## 何时使用此技能

当用户需要对以下内容进行深度评估时触发此技能：
- 产品方案或功能设计
- 业务战略或商业计划
- 技术架构或实施方案
- 流程优化或变革提案
- 投资决策或资源分配

**触发关键词示例：**
- "评估这个方案"
- "分析这个想法的可行性"
- "这个计划有什么风险"
- "从多个角度看看这个提议"
- "帮我全面评估一下"

## 评估框架

### 五个评估维度

对于每个输入的方案，依次从以下五个维度进行分析：

#### 1. 🔍 怀疑论者 (The Skeptic)

**角色定位：** 识别潜在风险与缺陷的批判性思考者

**评估重点：**
- 识别方案中的逻辑漏洞和假设缺陷
- 发现潜在的技术风险和实施障碍
- 指出可能被忽视的成本和代价
- 质疑乐观预期的合理性
- 挑战数据和证据的充分性

**输出格式：**
```
### 🔍 怀疑论者视角

**主要质疑：**
- [质疑点1]
- [质疑点2]
- [质疑点3]

**潜在风险：**
- [风险1及其影响]
- [风险2及其影响]

**关键假设挑战：**
- [假设1为什么可能不成立]
- [假设2的脆弱性]

**风险评级：** [低/中/高]
```

---

#### 2. 👥 用户拥护者 (The User Advocate)

**角色定位：** 关注用户体验与真实需求的代言人

**评估重点：**
- 评估方案对目标用户的真实价值
- 分析用户体验的流畅度和易用性
- 识别用户痛点是否得到有效解决
- 考察是否符合用户心智模型
- 评估可访问性和包容性设计

**输出格式：**
```
### 👥 用户拥护者视角

**用户价值分析：**
- 核心价值：[描述主要用户获益]
- 受益用户群：[明确目标用户]
- 痛点解决度：[评分及说明]

**用户体验评估：**
- 易用性：[分析学习曲线和操作复杂度]
- 体验流畅度：[识别摩擦点]
- 情感共鸣：[是否满足用户情感需求]

**用户需求匹配度：** [低/中/高]

**改进建议：**
- [建议1]
- [建议2]
```

---

#### 3. ⚙️ 执行者 (The Implementer)

**角色定位：** 分析落地可行性的实践专家

**评估重点：**
- 评估技术实现的复杂度和可行性
- 分析所需资源（人力、时间、预算）
- 识别实施过程中的关键依赖
- 评估团队能力和技术栈匹配度
- 考察交付时间的合理性

**输出格式：**
```
### ⚙️ 执行者视角

**可行性分析：**
- 技术复杂度：[低/中/高 + 说明]
- 实施难点：[列出主要挑战]
- 关键依赖：[技术/团队/外部因素]

**资源需求评估：**
- 人力：[所需人数和技能]
- 时间：[预估周期及关键里程碑]
- 预算：[成本估算和资源分配]

**实施路径建议：**
- 阶段划分：[分步实施计划]
- 优先级排序：[MVP vs 完整方案]
- 风险缓解：[如何降低实施风险]

**可行性评级：** [低/中/高]
```

---

#### 4. 🔮 远见者 (The Visionary)

**角色定位：** 评估长远价值与趋势的战略思考者

**评估重点：**
- 分析方案的长期战略价值
- 评估与未来趋势的契合度
- 识别潜在的创新机会和延展性
- 考察竞争优势和差异化定位
- 预测未来演进的可能路径

**输出格式：**
```
### 🔮 远见者视角

**长期价值评估：**
- 战略意义：[对组织长期目标的贡献]
- 市场前景：[趋势契合度和增长潜力]
- 竞争优势：[差异化价值和护城河]

**创新性分析：**
- 创新维度：[技术/模式/体验创新]
- 领先性：[行业定位和时机把握]

**可持续性：**
- 可扩展性：[规模化潜力]
- 适应性：[应对变化的灵活度]
- 延展机会：[未来可能的演进方向]

**战略价值评级：** [低/中/高]
```

---

#### 5. 🌐 系统思考者 (The Systems Thinker)

**角色定位：** 衡量对整体系统影响的全局观察者

**评估重点：**
- 分析方案对现有系统的影响
- 识别跨团队/跨部门的连锁效应
- 评估与其他计划的协同或冲突
- 考察技术债务和维护成本
- 分析生态系统层面的影响

**输出格式：**
```
### 🌐 系统思考者视角

**系统影响分析：**
- 对现有系统的影响：[兼容性/破坏性变更]
- 连锁效应：[上下游依赖和影响范围]
- 协同与冲突：[与其他项目的关系]

**整体性评估：**
- 技术债务：[是否引入新的复杂度]
- 维护成本：[长期运维负担]
- 组织影响：[流程/文化/结构变化]

**生态视角：**
- 平台效应：[对整体生态的贡献]
- 网络效应：[规模带来的价值增量]
- 外部影响：[对合作伙伴/用户生态的影响]

**系统健康度评级：** [低/中/高]
```

## 使用指南

### 输入要求

为获得高质量的评估结果，输入的方案应包含：
- **背景信息：** 问题背景、目标、现状
- **方案描述：** 核心内容、实施路径、预期成果
- **约束条件：** 资源限制、时间要求、技术限制

### 评估原则

1. **客观中立：** 每个维度独立分析，避免预设立场
2. **证据导向：** 基于方案提供的信息和合理推断
3. **建设性批评：** 识别问题的同时提供改进方向
4. **平衡视角：** 综合结论需权衡各维度发现
5. **可操作性：** 建议应具体、可执行

### 适应性调整

根据评估对象的特性，可灵活调整：
- **技术方案：** 加重"执行者"和"系统思考者"权重
- **产品设计：** 加重"用户拥护者"和"远见者"权重
- **战略决策：** 加重"远见者"和"系统思考者"权重
- **快速验证：** 可简化各维度的分析深度

## 输出示例

以下是一个完整评估的结构示例（内容简化）：

> **输入方案：** "构建AI驱动的客户服务智能问答系统"

### 🔍 怀疑论者视角
**主要质疑：**
- AI回答准确度如何保证？误答可能导致客户流失
- 知识库维护成本可能被低估
**风险评级：** 中

### 👥 用户拥护者视角
**用户价值分析：**
- 核心价值：7×24小时即时响应，提升用户体验
- 用户需求匹配度：高
**需求匹配度：** 高

### ⚙️ 执行者视角
**可行性分析：**
- 技术复杂度：中（需要NLP模型训练和集成）
- 预估周期：3-4个月MVP
**可行性评级：** 中

### 🔮 远见者视角
**长期价值评估：**
- 战略意义：符合客服数字化转型趋势
- 市场前景：AI客服是行业标配
**战略价值评级：** 高

### 🌐 系统思考者视角
**系统影响分析：**
- 对现有系统的影响：需与CRM、工单系统集成
- 维护成本：知识库需持续更新
**系统健康度评级：** 中

## 📊 综合评定
**总体评价：** 推荐

**决策建议：**
- 如果采纳：优先构建MVP验证核心场景，建立知识库更新机制
- 优化路径：先覆盖高频问题，逐步扩展至复杂场景

---

## 注意事项

1. **避免过度批判：** 怀疑论者视角应建设性，而非一味否定
2. **保持独立性：** 各维度分析应独立进行，避免相互影响
3. **结合上下文：** 评估应考虑组织文化、行业特点、发展阶段
4. **动态调整：** 根据方案的复杂度和紧急度调整分析深度
5. **明确局限性：** 评估基于提供的信息，信息不足时应明确指出
